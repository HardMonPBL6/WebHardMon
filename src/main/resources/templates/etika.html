Etika eta Jokabide Profesionala

HardMon proiektua · Hardwarearen monitorizazio-zerbitzu digitala

Erreferentziak: ACM Code of Ethics and Professional Conduct (2018) · CCII kode deontologikoa

1. Aurkezpena eta testuingurua

HardMon-ek zerbitzu digital bat eskaintzen die bezero-enpresei: haien ekipo informatikoen hardwarearen egoera eta errendimendua etengabe monitorizatzea, matxurak aurreikusteko eta ekipoen bizitza erabilgarria luzatzeko. Horretarako, bezeroaren langileen ordenagailuetan agente arin bat exekutatzen da, eta hardware-metrikak (PUZaren karga, tenperatura, memoria- eta disko-erabilera, eta abar) nahiz makina/erabiltzaile identifikatzaile bat jasotzen ditu. Telemetria hori azpiegitura hibrido batera bidaltzen da (GCP hodei publikoa eta Cloud lokala, Cloudflare segurtasun-geruzak babestuta), non balioztatu, prozesatu, StressScore izeneko adierazle batez aberastu eta biltegiratzen den; azkenik, bezeroari kontrol-panel baten bidez aurkezten zaizkio emaitzak.

Zerbitzu honek hirugarren pertsonen ekipoetatik datozen datuak tratatzen dituenez —eta, bereziki, lan-harreman baten testuinguruan—, gure ardura etikoa areagotu egiten da. Langileen ekipoen monitorizazioak datu pertsonalak, intimitatea eta enpresaren eta langilearen arteko botere-asimetria ukitzen ditu zuzenean. Hori dela eta, jardunbide profesional honek erreferentzia gisa hartzen ditu ACM-ren etika-kodea eta CCII-ren kode deontologikoa, eta gure testuinguru zehatzera egokitzen ditu.

2. Printzipio etikoak

ACM Code of Ethics-en oinarrizko atalei (1. eta 2. ataletako printzipio orokorrei eta lanbide-ardurei) lotutako printzipioak

ACM 1.1

2.1. Gizartearen eta pertsonen ongizatearen aldeko ekarpena

Gure helburua bezeroari balio erreala ematea da: matxurak aurreikusiz baliabideak aurreztea eta ekipoen bizitza luzatuz hondakin elektronikoak murriztea. Konpromisoa hartzen dugu sistema xede horretarako erabiltzeko, eta ez langileak presionatzeko edo haien gainean kontrol gehiegizkoa ezartzeko tresna gisa. StressScore adierazlea hardwarearen estresa neurtzeko diseinatu da, ez pertsonen errendimendua epaitzeko.

ACM 1.2

2.2. Kalterik ez eragitea

Monitorizazioak kalte sor diezaieke langileei, datuak haien aurka erabiliz gero (zaintza, sailkapenak, zigorrak). Kalte hori prebenitzeko, neurri teknikoak eta antolakuntzazkoak hartu ditugu: bildutako datuak hardwarera mugatzen dira (datu-minimizazioa), eta ez dugu erabiltzailearen jardueraren edo edukien jarraipenik egiten (ez teklatua, ez nabigazioa, ez pantaila).

ACM 1.3

2.3. Zintzotasuna eta fidagarritasuna

Garatzen dugun sistemaren mugak eta gaitasunak modu gardenean azaltzen ditugu. Erabilgarritasun-probetan eta emaitzetan ez dugu inolako daturik faltsutzen edo apaintzen: emaitza errealak baino ez ditugu erabiltzen erabakiak hartzeko, eta horixe da gure HMI ebaluazioaren oinarria.

ACM 1.4

2.4. Justizia eta diskriminaziorik eza

Sistemak langile guztiak modu berean tratatu behar ditu, eta ezin du diskriminaziorako edo banakako sailkapenetarako bide eman. Horregatik, panel nagusietan datuak ekipo- edo talde-mailan agregatuta aurkezten dira lehenetsita, eta banakako ikuspegia sarbide-kontrol zorrotzaren mende dago. Era berean, zerbitzua pertsona guztientzat eskuragarri egotea bilatzen dugu, irisgarritasuna bermatuz.

ACM 1.6

2.5. Pribatutasunaren errespetua

Pribatutasuna gure diseinuaren printzipio gidaria da (privacy by design). Datu-minimizazioa aplikatzen dugu, makina/erabiltzaile identifikatzaileak pseudonimizatzen ditugu ahal den neurrian, eta atxikitze-epe mugatuak ezartzen ditugu. Datu pertsonalen tratamendua Datuak Babesteko Erregelamendu Orokorrera (DBEO/RGPD) egokitzen dugu, oinarri juridiko egokia eta gardentasun-betebeharrak errespetatuz.

ACM 1.7

2.6. Konfidentzialtasuna

Bezeroaren datuak —telemetria, lizentzia-informazioa eta enpresaren konfigurazioa barne— konfidentzialak dira. Sarbidea behar dutenei baino ez zaie ematen, eta ez ditugu hirugarrenei lagatzen edo helburu komertzialetarako berrerabiltzen. Lizentzia balioztatzeko prozesuan tratatzen den informazioa ere xede horretara mugatzen da.

ACM 2.1–2.2

2.7. Kalitate eta gaitasun profesionala

Lan profesionala kalitate handiz egitea gure ardura da. Kodearen segurtasun-auditoretzak egiten ditugu tresna espezializatuekin (SonarQube, Snyk edo Semgrep), antzemandako akatsak zuzentzen ditugu, eta garapen-jardunbide egokiak definitzen ditugu akatsak berriz ager ez daitezen. Halaber, karga- eta estres-probak egiten ditugu osagai kritikoenen gainean.

ACM 2.5–2.9

2.8. Ebaluazio sendoak eta arriskuen kudeaketa

Sistema osoaren ebaluazio sendoak egiten ditugu, arriskuen analisi formal batetik abiatuta. Sistema fidagarriak eta seguruak diseinatzen ahalegintzen gara, eta arrisku etiko nahiz teknikoak modu sistematikoan identifikatu, aztertu eta arintzen ditugu.

3. Segurtasuna, pribatutasuna eta datuen gobernantza

Printzipio etikoak babesteko eta betetzeko hartu ditugun neurri konkretuak

3.1. Segurtasun-neurriak

  • Hodeien arteko komunikazioa WireGuard bidez zifratuta (IP publikoaren gainetik ere)
  • Cloudflare tunel autentifikatua telemetria sarrerarako (HTTPS + Access token)
  • Garraioko (TLS) eta atsedeneko zifratzea, suebakiekin eta sare-segmentazioarekin
  • Ostalarien gogortzea: SSH root desgaituta, gako publiko bidezko autentifikazioa, fail2ban/CrowdSec
  • Babeskopia-politika eta hondamendien aurreko berreskuratze-plana egiaztatua
  • Fitxategi garrantzitsuen osotasuna kontrolatzeko sistema (AIDE/Tripwire)

3.2. Pribatutasun-neurriak

  • Datu-minimizazioa: hardware-metrikak baino ez; ez jarduera-, ez edukien, ez kokapenaren jarraipenik
  • Makina/erabiltzaile identifikatzaileen pseudonimizazioa identitate erreala babesteko
  • Atxikitze-epe mugatuak eta datuak ezabatzeko prozedurak automatizatuta
  • Gardentasuna: langileei jakinaraztea zer datu biltzen diren, zertarako eta zenbat denboraz
  • DBEO/RGPD betetzea: oinarri juridikoa, eskubideak eta prozesatzaileekiko bermeak

3.3. Datuen gobernantza

  • Sarbide-kontrola rol bidez: ikuspegi agregatua lehenetsita, xehetasunerako baimen espezifikoa
  • NiFi-ren jatorri-erregistroa (data provenance) aktibatuta datuen ibilbidea dokumentatzeko
  • Lizentzien eta sarbideen kudeaketa zentralizatua, NiFi-k DB-aren kontra balioztatuta
  • Datuen erantzukizunaren banaketa argia bezeroaren eta zerbitzu-hornitzailearen artean
